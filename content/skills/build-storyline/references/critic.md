# Storyline Critic — Self-Review Gate

Review the storyline the way a McKinsey engagement manager would at 2am before a client meeting. Grade it, review each section, give the top 3 fixes, and name one thing that works.

## Grading Scale

| Grade | Meaning |
|---|---|
| **Green** | Ready to present. Minor polish only. |
| **Yellow** | Fixable overnight, but do not present as-is — structural gaps undermine credibility. |
| **Red** | Needs rework. Core argument or flow is broken. |

## What to Check

**Structure**
- Does it flow problem → context → analysis → solution?
- Does it answer the SCQA Question and land on the Answer?
- Clear beginning, middle, end; section breaks in the right place?

**Rigor**
- Every claim backed by data or a source?
- Numbers specific, not vague (`grew significantly` → `grew 25% YoY`)?
- Comparisons fair and complete; competitors named, not generic?
- A financial frame where one is needed?

**So-What**
- Each slide carries a takeaway, not just data?
- Titles-only read gives the full argument (ghost-deck test)?
- Recommendation actionable and time-bound (owners + dates)?

**Completeness**
- Comparison framework where options are presented (not a bare list)?
- Next steps specific with owners and timelines?
- No illogical jumps between adjacent slides?

## Output Format

```
# Storyline Critic: [Title]

**Grade: [GREEN/YELLOW/RED] — [one-line verdict]**

## Section-by-Section
| Slide / Section | Status | Issue |
|---|---|---|
| [Title] | Green/Yellow/Red | [specific issue or "Strong"] |

## Top 3 Fixes
### Fix 1: [Title]  — [what's wrong + exactly how to fix]
### Fix 2: [Title]  — [what's wrong + exactly how to fix]
### Fix 3: [Title]  — [what's wrong + exactly how to fix]

## Keep This
[the one thing that works — so it's protected in the rewrite]
```
