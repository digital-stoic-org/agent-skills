# Storyline Templates

Each line becomes one slide title. Replace placeholders; add/remove lines to fit the story; keep the logical progression.

## 1. Pitch / Market Strategy

Flow: market opportunity → competitive position → product strategy → go-forward plan.

```
1. [Market] represents $XXB opportunity growing at XX% CAGR
2. We operate in [segment] worth $XXB with XX% growth
3. Top 3 competitors generate $XXM-XXB revenue growing XX-XX% annually
4. Our revenue of $XXM positions us as [rank] with XX% growth
5. [Product] addresses [use case] for [target segment]
6. Top 10 customers span [sectors], XX% enterprise vs XX% SMB
7. Pricing structured as [model] with $XX average contract value
8. Product differentiation built on [technology/approach] vs competitors
9. Key competitive advantages: [adv 1], [adv 2], [adv 3]
10. Three growth opportunities: [opp 1], [opp 2], [opp 3]
11. Focus on [priority opp] based on market size + competitive position
12. 18-month roadmap prioritizes [cap 1], [cap 2], [cap 3]
```

## 2. Problem-Solving (Issue Tree)

Flow: problem framing → root-cause → solution options → prioritization → next steps.

```
1. [Problem] — current state at XX vs target of XX
2. Problem driven by three factors: [f1], [f2], [f3]
3. [Factor 1] contributes $XXM impact (XX% of total)
4. [Factor 2] contributes $XXM impact (XX% of total)
5. [Factor 3] contributes $XXM impact (XX% of total)
6. Root-cause analysis reveals [key insight from data]
7. Three solution approaches identified to address root causes
8. Solution 1: [approach] — XX% impact, $XXM investment, XX weeks
9. Solution 2: [approach] — XX% impact, $XXM investment, XX weeks
10. Solution 3: [approach] — XX% impact, $XXM investment, XX weeks
11. Prioritize [solution X] based on impact/effort analysis
12. Roadmap: [Phase 1 by date], [Phase 2 by date], [Phase 3 by date]
13. Success metrics: [m1], [m2], [m3] tracked [frequency]
```

## 3. Roadmap / Implementation Plan

Flow: approach → phases → activities → timeline → success criteria.

```
1. Objective: [goal with measurable outcome]
2. Four-phase approach over XX weeks: Discovery → Design → Build → Launch
3. Roadmap spans XX weeks with owners and milestones:
   - Phase 1 (Wk 1-X): User research — XX interviews across [segments] | Owner: [role]
   - Phase 2 (Wk X-X): Solution design — stories, sprint planning | Owner: [role]
   - Phase 3 (Wk X-X): MVP build — [features] across XX sprints | Owner: [role]
   - Phase 4 (Wk X-X): Launch — onboard XX customers | Owner: [role]
4. Core team of XX across [# domains] — Gap: need [X roles] + $XXK
5. Success criteria: [m1] = XX, [m2] = XX by [date]
6. ROI: track [business metric] over XX months
```

## Reusable Arcs (pick when no template fits)

```
Problem-to-Solution:  Problem → Sizing → Root causes → Options → Recommendation → Plan
Market-to-Strategy:   Opportunity → Landscape → Our position → Strategy → Roadmap → Outcomes
Analysis-to-Action:   Key question → Hypotheses → Data → Insights → Recommendations → Next steps
```

## Effective-Storyline Tips

- **Start with structure** — outline sections first, then fill slides. 15–25 slides for an exec deck.
- **Exec summary upfront** — one slide with problem + recommendation + expected impact, so execs get the punchline immediately.
- **Parallel structure** — sibling slides in identical format (easier to follow).
- **Signposts** — "Three drivers of the problem" followed by exactly three slides.
- **Build to the punchline but don't bury the lede** — lead them through the logic; still state the answer early.
- **Iterate the storyline, not the slides** — far cheaper to reorder titles than finished slides.
